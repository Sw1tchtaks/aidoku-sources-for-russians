# RanobeHub (ru.ranobehub)

Источник для [ranobehub.org](https://ranobehub.org) — русскоязычная платформа лайт-новелл.

## Реализация

В отличие от [Ranobes](../ru.ranobes/), здесь чистый JSON REST API:

- **Каталог**: `GET /api/search?page=N` → 12 элементов на страницу + `pagination.{currentPage,lastPage}`
- **Поиск**: `GET /api/fulltext/global?query=…` — общий fulltext, выбираю секцию `ranobes`. Один запрос — нет пагинации.
- **Карточка**: `GET /api/ranobe/{id}` → `{data: {...}}` с обложкой (`posters.{big,medium,small}`), названиями (`names.{rus,eng}`), авторами/переводчиками, статусом, тегами/жанрами.
- **Список глав**: `GET /api/ranobe/{id}/contents` → массив томов, каждый с `chapters[]`. Развёртываю в плоский список и переворачиваю (новейшие сверху).
- **Текст главы**: `GET /api/chapter/{id}` → `chapter.text` (HTML), пропускаю через свой strip-тегов в markdown.

## Настройки

- **Английские названия** — переключает `names.eng` ↔ `names.rus` для тайтлов и тегов.

## Известные ограничения (v1)

- Нет фильтров по жанрам/статусу/году. `filters.json` пуст.
- Поиск не пагинируется (API такого не даёт через найденный endpoint).
- Парсер HTML→markdown простой: убирает теги, заменяет `<p>`/`<br>` на переводы строк, декодирует базовые entities. Сложное форматирование (таблицы, списки) теряется.
- Картинки внутри текста главы (`<img>`) удаляются. Главы только-картинки (`has_images: true` без текста) останутся пустыми.
- Платные/спонсорские главы (если у RanobeHub такие есть) пока не помечаются как locked.
