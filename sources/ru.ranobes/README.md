# Ranobes (ru.ranobes)

Источник для [ranobes.com](https://ranobes.com) — переводы лайт-новелл и веб-новелл на русский.

## Реализация

DLE-сайт, API нет, всё парсится из HTML. Это **первый текстовый (не картиночный) источник** в репозитории — главы возвращаются как `PageContent::Text(markdown)` и Aidoku рендерит их в вертикальном тексто-ридере (`Viewer::Vertical`).

- **Каталог**: `GET /ranobe/page/N/` → 10 тайлов на страницу
- **Поиск**: `GET /index.php?do=search&subaction=search&story=…` (DLE-стандарт)
- **Карточка**: `GET /ranobe/{id}-{slug}.html` — заголовок из `h1.title`, описание из `meta[og:description]`, обложка из `meta[og:image]`
- **Список глав**: `GET /chapters/{slug}/page/N/` (slug — другой, ранобэс обрезает его), 25 глав на страницу. Источник идёт по страницам до `CHAPTER_LIST_PAGES_CAP=50` (≈1250 глав максимум).
- **Текст главы**: `GET /chapters/{slug}/{id}-{page}.html`, парсит `div#arrticle p` (да, "arrticle" с двумя r — типо в DLE-шаблоне) и склеивает абзацы через двойной перевод строки. Возвращает один `Page` с этим текстом.

## Настройки

- **Веб-домен** — переключается, если основной домен отдаёт ошибки/каптчу.

## Известные ограничения

- Нет фильтров по жанрам/статусу (`filters.json` пустой). Каталог сортируется по дефолту сайта (новейшее).
- Длинные новеллы (600+ глав) делают много HTTP-запросов на refresh — каждая страница списка = 1 запрос. При rate-limit 2/сек это до ~30 секунд ожидания.
- Главы из нескольких частей (`-2.html`, `-3.html`) не объединяются — каждая часть отдельная глава в списке. Если ranobes.com разбивает один текст на несколько URL, это будет видно как `Глава X (1/3)`, `Глава X (2/3)`...
- Даты парсятся по русской локали "21 апреля 2026 в 21:37". Если форматирование сменится — даты глав станут пустыми, но чтение продолжит работать.
- Парсер статуса манги пока ищет по `r-fullstory-spec` ключевые слова ("завершён"/"выпускается"/"заморожен"), могут быть пропуски.
