# Ранобэ.рф (ru.ranoberf)

Источник для [ранобэ.рф](https://xn--80ac9aeh6f.xn--p1ai) — Cyrillic-домен русскоязычного сайта лайт-новелл (punycode `xn--80ac9aeh6f.xn--p1ai`).

## Реализация

Сайт построен на Next.js. Один полу-API endpoint и Next.js `__NEXT_DATA__`-скрейпинг для остального:

- **Каталог**: `GET /v3/book` — возвращает **сразу все 800 книг** одним JSON (1.1 МБ). Параметры пагинации (`offset/limit/page/skip/take/per_page`) игнорируются. Источник скачивает целиком, режет на страницы по 30 элементов на стороне приложения.
- **Поиск**: фильтрует загруженный каталог по подстроке в заголовке. Не идёт на сервер отдельно.
- **Карточка**: `GET /{slug}` — обычная HTML-страница, парсится `<script id="__NEXT_DATA__">` JSON, читается `props.pageProps.book` (структура из ~30 полей: title, slug, status, author, genres, verticalImage, chapters, …). Cover берётся из `book.verticalImage.url` (URL-name типа `vertical-85.jpeg` где номер версии загрузки уникален для каждой книги — поэтому в каталоге обложки нет, она появляется после открытия карточки).
- **Список глав**: уже встроен в `book.chapters` (массив до ~2000 элементов). Никаких дополнительных запросов.
- **Текст главы**: `GET {chapter.url}` (URL уже относительный, типа `/against-the-gods/glava-1858-boynya`), парсится `__NEXT_DATA__`, читается `chapter.content.text` — HTML, прогоняется через strip-тегов.

## Известные ограничения (v1)

- **Без обложек в каталоге** — единственный способ получить cover это открыть страницу книги. У карточки обложка уже будет.
- **Каталог дорогой** — 1.1 МБ на любой refresh. Aidoku может закешировать, но пока этого нет — каждое открытие источника тащит весь список.
- **Нет фильтров по жанрам/статусу** — `filters.json` пуст. Поиск только текстовый.
- **Платные/донат-главы** помечаются `locked = true` в Aidoku когда `isDonate || isSubscription` и пользователь не платил. Отдельного логина в источнике нет — авторизация на сайте не реализована.
- **Punycode-домен** — все запросы идут на `xn--80ac9aeh6f.xn--p1ai` (форма ASCII). Для пользователя в Aidoku отображается так же, кириллический алиас не используется.
- HTML-стрипер убирает `<div class="messageBlock">` (служебные баннеры на страницах глав), но сложное форматирование вроде таблиц/списков теряется.
