# Senkuro (ru.senkuro)

Источник для [senkuro.com](https://senkuro.com) — современная платформа манги, ранобэ и комиксов на русском.

## Реализация

Senkuro и Senkognito работают на одном GraphQL API (`https://api.senkuro.me/graphql`). Общий движок вынесен в шаблон [`templates/senkuro`](../../templates/senkuro), а сам этот источник — тонкая обёртка, задающая `BASE_URL` и список постоянно скрываемых тегов (18+ исключаются всегда, как в Tachiyomi-расширении keiyoushi).

Используются Tachiyomi-нейм-операции, доступные в публичной схеме:

- `searchTachiyomiManga` — каталог + поиск (тип/формат/статус/перевод/возраст в фильтрах)
- `fetchTachiyomiManga` — карточка манги
- `fetchTachiyomiChapters` — список глав вместе с командами переводчиков
- `fetchTachiyomiChapterPages` — страницы главы

Если схема Senkuro изменится — править нужно в `templates/senkuro/src/graphql.rs` (одно место для обоих источников).

## Настройки

- **Домен API** — переключение между `api.senkuro.me` (россия) и `api.senkuro.com` (публичный).
- **Английские заголовки** — показывать английские варианты названий и тегов вместо русских.

## Известные ограничения

- Жанры пока не подтягиваются динамически (в Tachiyomi их подгружает `mangaTachiyomiSearchFilters` при первом открытии каталога). В фильтрах есть только тип/формат/статус/статус перевода/возраст. Жанры можно добавить отдельной задачей — потребуется блокирующий запрос к API при `get_search_manga_list` либо реализация `DynamicFilters`.
- Глубокие ссылки (`/manga/<slug>` без ID) технически отдают `key = ",,slug"`. Для прямых внешних линков ID-часть не известна заранее — Aidoku запросит детали и упадёт. Открытие манги через каталог работает нормально, потому что ключ уже в форме `id,,slug`.
